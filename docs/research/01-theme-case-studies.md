- **Reviewer 通過條件**：明顯通過（強證據 + 強邏輯）

- **持有週期**：6-12 個月

#### 因果樹

```

├── 一階：航運直接受益

│   ├── 造船：台船(2208)
│   ├── 貨櫃製造（無台股代表，國際標的）
│   └── 反應時間：T+3 到 T+6 月

    ├── 受害方：航空貨運漲價、進口商成本壓力
    └── 反應時間：T+6 月+
```

#### 可偵測性評估
- **如果系統當時存在，能不能抓到？** ⚠️ **要看系統有沒有追運價數據**
  - 純看新聞 → ❌ 抓不到（新聞登場時已晚）
  - 有追 SCFI / BDI 等指數 → ✅ 能在 2020/8-10 提早提報
- **關鍵訊號**：SCFI 連續 4 週上漲 ≥ 20%，且 BDI 同步走高
- **歷史對應**：2003-2004 SARS 後復甦曾有類似運價飆漲，但規模較小（Reviewer 可引用為弱證據）
- **Reviewer 通過條件**：
  - 強證據（數據趨勢明確）✅
  - 邏輯鏈（疫情不均衡復甦 → 集裝箱錯位 → 運價飆漲）✅
- **系統需要的能力**：**必須有公開數據追蹤模組，不能只靠 LLM 讀新聞**

## Step 4：研究結果反推系統設計

### 4.0 Scout 的 5 種感測器架構（2026-05-31 補上 Policy + Roadmap）

通過 Arthur 的歸納反思，發現所有觸發路徑最終會匯流到 **族群識別 → 因果樹 → 歷史對應（optional）**，但**初始觸發源有 5 種**：

```
觸發源 (5 種感測器):
├── News       ── 新聞異常突起（中英文）          ★ 口罩、國防、AI 第一波
├── Price      ── 族群漲停潮 / 異常量（safety net）★ 漏抓回溯機制
├── Data       ── 公開數據異常（運價、原物料）     ★ 航運、銅
├── Policy     ── 政策/法規公告                  ★ 高端疫苗 EUA、川普關稅
└── Roadmap    ── 大廠路線圖/法說會規格升級       ★ 矽光子、HBM 規格

                      ↓
            [族群識別 + 因果樹展開到三階以上]
                      ↓
        [Reviewer 推理引擎 — 統一邏輯]
        ├── 歷史對應強 → 強訊號（直接列入今日報告）
        ├── 歷史弱但邏輯強 → 觀察清單（追蹤但不推薦）
        └── 兩者都弱 → 駁回 + 記錄理由（負樣本學習）
                      ↓
                  [今日報告 + 觀察清單]
```

**重要設計決策：**

1. **Price Driven 是「safety net」，不是平行觸發**：當 Price 觸發時，代表 News/Data/Policy/Roadmap 漏抓了領先訊號。Reviewer 必須**主動回溯 30-90 天**，把漏抓的訊號加入「**漏抓案例庫**」，供 Scout 改進。
2. **Policy 需要獨立爬蟲**：政策公告埋在法規網站、政府公報、健保署網頁等，**不會出現在新聞 RSS**，需要單獨的 collector。
3. **Roadmap 領先股價 6-12 個月**：法說會規格升級的影響會延後反映，但**訊號早**，是長波段題材的最佳早期指標。
4. **歷史對應是 optional**：對全新題材（COVID、ChatGPT）強邏輯推論足以通過審查，降級為「觀察清單」。

### 4.1 系統必須具備的訊號源（按重要性排序，對應 5 sensors）

| 訊號源 | 屬於哪個 sensor | 抓到的案例 | 優先級 |
|---|---|---|---|
| **中文新聞**（經濟日報、鉅亨、自由）| News | 口罩、國防、AI 第一波 | P0 |
| **國際新聞（英文）**：Reuters、Bloomberg、CNBC | News | 俄烏、ChatGPT、TSMC、NVIDIA | P0 |
| **公開財經數據**：SCFI/BDI、LME 銅價、DRAM 報價 | Data | 航運、銅、矽晶圓 | P0 |
| **台廠法說會逐字稿**：MOPS 公開資訊觀測站 | Roadmap | CoWoS、AI 二階、矽光子 | P0 |
| **政府/法規公告**：金管會、衛福部、健保署、行政院 | **Policy** | 高端 EUA、太空法、健保藥價 | **P0**（新加） |
| **國際科技公司路線圖**（NVIDIA, AMD, TSMC 法說）| Roadmap | AI、矽光子、機器人 | P0 |
| **市場行情**：各族群漲停數、異常成交量 | Price | safety net（漏抓回溯）| P0 |
| **Google Trends**：主題熱度判定 | News (輔助) | 所有新聞驅動題材 | P1 |
| **PTT/Dcard 股板**：散戶情緒 | News (輔助) | AI 散戶湧入、口罩情緒 | P1 |
| **產業聯盟/協會公告**：SEMI、TAITRA | Roadmap / Policy | 矽光子 SEMI 聯盟 | P2 |

→ **第一版系統至少要覆蓋 P0 的 7 個訊號源**（News 中英 + Data + MOPS 法說 + Policy + 國際路線圖 + 市場行情），對應 5 sensors 全覆蓋。

### 4.2 Scout Agent 偵測範圍

每日掃描內容（Scout 不下判斷，只做「異常標記」）：

1. **新聞異常**：相對前 7 天，某主題詞 / 公司名 / 國家 / 產業出現頻率突起 ≥ 2x
2. **Google Trends 異常**：搜尋量單日突起、24 小時環比 ≥ 50%
3. **數據異常**：追蹤約 30 個公開指數，任一週環比 ≥ 5% 或月環比 ≥ 15%
4. **法說會關鍵詞**：「缺貨」「供應吃緊」「漲價」「擴產」「新規格」「合作」+ 公司名
5. **跨市場聯動**：美股某類股一週漲幅 ≥ 10%，自動標記台股對應族群

Scout 預期每日輸出 5-15 個候選主題（多數是雜訊，這就是 Reviewer 的工作）。

### 4.3 Reviewer Agent 必備能力

從 5 個案例反推 Reviewer 必須能做：

1. **歷史庫**：至少預先建好這 20 個題材的「事件 → 受益股 → 反應時間」資料庫，能做案例對應
2. **因果樹推理**：對任何候選主題，能推導 一階/二階/三階 受益股 + 各階反應時間
3. **訊號分級**：「強訊號 → 直接列入今日報告」「中訊號 → 觀察清單」「弱訊號 → 駁回但記錄」
4. **跨市場關聯**：知道美股 NVIDIA 是台股緯創 的領先指標
5. **供應鏈知識**：知道 ABF 載板是筆電的關鍵零件
6. **承認新型題材**：對沒歷史對應的事件（如生成式 AI），用強邏輯通過審查並標記「無歷史對應，謹慎」

### 4.4 系統的關鍵風險（必須在設計時處理）

1. **數據訊號 vs 新聞訊號的延遲差**：航運股案例顯示數據先行 3-6 個月，系統不能只在「新聞登場」才提報
2. **歷史過度擬合**：Reviewer 若只認歷史對應，會漏掉 ChatGPT 這類全新事件
3. **三階以上的雞犬升天**：散戶湧入末段（如 2023/7 群創華映 沾 AI 邊）系統應警示「散戶 FOMO 階段，慎入」而非追進
4. **同名異質題材**：「AI 概念股」2017、2023 是不同的東西，歷史庫要時間分段

## 給 Arthur 的關鍵發現

1. **系統訊號源必須包含「公開數據」模組**（運價、銅價、DRAM 報價等）——只靠新聞會漏掉 1/3 的大波段
2. **訊號源從「中文新聞」往「國際新聞」+「國際大廠財報」延伸**——AI/矽光子等題材源頭都在國際
3. **Reviewer 的因果樹要分到三階以上**——只想到一階會漏掉真正的大利多（2021 ABF 是宅經濟的孫子輩）
4. **5 個案例中，4 個可被當時系統提早抓到**（口罩、AI、國防、矽光子），航運要看是否有追數據——可行性 **80%**
5. **歷史庫建構是 Reviewer 的核心能力**：建議先把這 20 個題材的「事件 + 受益股 + 反應時間」結構化，當作系統第一版的「先驗知識」
