# PolyDig 優化建議 —— 金融分析師視角(賺錢價值 × 金融專業)

**作者視角**:買方分析師 / 量化(CFA-type)
**日期**:2026-06-01 ｜ **版本**:v2(經 `finance-reviewer` agent 審查後修訂,評分 B− → 見文末修訂紀錄)
**定位**:跟 VC 報告(談募資/估值)不同,這份談**訊號本身有沒有可實現的 alpha,以及怎麼把訊號變成錢**。

> 一句話:**PolyDig 找到的是「題材的進場時機」,但 ① 它宣稱的漲幅多半「進場價根本不存在」(開盤漲停買不到)、② 沒做 beta 調整(高 beta 題材的「超額」可能大半是市場 beta)、③ 只有進場、沒有出場/部位/風控。** 在證明 net alpha 之前,任何變現都是空中樓閣。

---

## 1. 核心問題:這訊號有沒有「可實現(net)」的 alpha?

Phase 0 算的是**毛(gross)、後見之明選股、收盤價假設**的超額報酬。機構級會立刻追問,且每一項都比我原本寫得更嚴重:

| 追問 | 現況(已核對程式碼) | 嚴重性 |
|---|---|---|
| **🔴 訊號日買得到嗎?** | validator 用 **T-1 收盤**進場([excess_return.py:113](src/polydig_validator/excess_return.py:113));但康那香等小型股在訊號後**首個交易日即開盤漲停(±10% 鎖死)**,T-1 收盤的對應買單**根本不存在**。 | **不是「打折」,是「無效」**——+310% 的基礎在多數漲停案例中不成立 |
| **🔴 beta 調整?** | 只減 TAIEX,**無 beta**([excess_return.py:119,159](src/polydig_validator/excess_return.py:119))。航運股 beta 約 1.5–2.5,若 beta=2、TAIEX 同期 +50%,光 beta 就貢獻 +100%。 | 高 beta 題材(航運/AI)**真實 alpha 可能只剩帳面的 30–70%** |
| 交易成本 | 未計(手續費 0.1425%×2 + 證交稅 0.3% + 滑價) | 短週期題材(口罩 1-3 月)吃成本重 |
| 流動性(非線性) | 未計;康那香當時日成交僅數千萬,**100 萬大單即顯著推價** | 訊號**完全不可規模化**(個人小額尚可,機構無法用) |
| 體制(regime) | 未做 | 多頭驗證的訊號在空頭是否仍有效?**這是 alpha 真實性的核心,不是優化項** |

→ **結論(重新框架)**:Phase 0 的「4/5 強領先 → GO」是 **selection-on-the-dependent-variable**(以結果挑樣本,5 個全是台股史上最戲劇化的爆發題材)。統計上**不能推導任何前瞻機率**。「GO」的正確意義是:「**評測管線能跑、計算邏輯正確**」,**不是**「訊號有可實現 alpha」。任何前瞻測試之前,+310%/+436.8% 都不是 alpha 證據。

## 2. 價值量化框架(把訊號變成可比較的數字)

```
EV(訊號) = 命中率 × 平均報酬(net) − (1−命中率) × 平均虧損(net) − 交易成本
   其中「命中率」分母 = 系統發出的訊號總數(不是只看贏家);目前此分母未知、無前瞻資料
領先價值 = 領先天數 × 該題材日均超額報酬(net、beta 調整後)
```
- 對標 benchmark:**買進持有 TAIEX** + **等新聞主流化才進場**(證明「早」真有溢價)。
- **風險調整**:算 Sharpe(需定義無風險利率 + 持倉假設)與**最大回撤**——題材股波動極大,只看漲幅會嚴重誤導。

## 3. 從「訊號」到「報酬」的缺口(最大的金融漏洞)

系統**只回答「何時進場」,完全沒回答**:

| 缺口 | 為何致命 | 優化 |
|---|---|---|
| **出場訊號** | 題材「漲快跌也快」;**跌法比漲法暴力(量縮急殺)**,沒出場 = 不是少賺、是**全吐**。康那香 T+180 的 +310% 帳面,中途歷經 -30~-50% 回撤 | 退場規則:熱度反轉、數據轉弱、漲幅達歷史中位數、time-stop、最大回撤停損 |
| **部位大小** | 全壓單一題材 = 賭博 | 波動度反比 / Kelly 上限 / 單一題材上限 |
| **風險控管 + 題材共線** | 多檔「AI 二三階」相關係數近 1,組合分散度極低 | 停損、**跨題材**(非同題材內)分散、總曝險上限 |
| **進場分批(漲停對策)** | 一階開盤漲停買不到 | 若 T 日一階漲停 → **同日改布局二階(lag 受益股)**,或待首波主力跌停解鎖後分批 |

> 這呼應系統獨有的 `lag_days` 設計:**用 lag 做二三階輪動**(一階擁擠買不到 → 提前布局二三階,如 ABF 載板、重電)——這是最可能規模化、賠率最好、且**能繞開漲停**的真 edge。

## 4. 訊號品質的金融專業優化

| 優化 | 金融理由 |
|---|---|
| **多重比較校正** | Phase 0 = 5 案例 × 3 觸發 × 多窗口 ≈ **45 次假設檢定**;即使全是雜訊,在無校正下「至少 2-3 個顯著」是統計必然。需 White's Reality Check / Deflated Sharpe |
| **體制過濾(已上移至 §1)** | 見上,屬 alpha 真實性核心 |
| **beta / 因子調整** | excess return 須做 CAPM 殘差或 Fama-French 三因子,才能分離真 alpha 與 beta/規模因子 |
| **流動性過濾** | 日成交量過低者標「不可規模化」,別推薦買不到的股 |
| **look-ahead 檢查** | cases.json 觸發日多為「事後確認節點」(如 1/20 人傳人);真實即時情境下,確認前媒體解讀分歧,訊號是否可在確認前辨識需另證 |

## 5. 變現(金融專業視角,非募資)

| 模式 | 說明 | 法規 / 順序 |
|---|---|---|
| **賣鏟子(B2B 資料/API 授權)** | 把「事件→因果樹→受益股」資料層賣給投顧/媒體/量化 | ✅ **法規風險最低、應為第一步** |
| 訂閱訊號服務 | 付費題材報告 | 🔴 高法律風險(見下) |
| 代操 / 全委 | 用訊號管帳戶 | 需投信投顧執照 |
| 自營/自用 | 自己用訊號交易 | ✅ 無對外薦股問題,但**先證明 net alpha** |

**🔴 投顧法量化邊界(非法律意見,務必諮詢律師)**:依《證券投資顧問事業管理規則》第 3 條,對**不特定多數人**提供「具體個別有價證券」的投資意見(明確到 ticker 的推薦)須持牌。**即使加「僅供參考、非投資建議」免責,只要標的明確到 ticker 且有付費訂閱,金管會仍可能認定為事實上的投顧服務**,違規最高罰 NT$240 萬並得停業。→ **獲牌或律師正式意見前,任何對外付費訊號服務都應視為高風險;第一步走 B2B 資料/API 授權。**

## 6. 優先序

| 優先 | 項目 | 金融邏輯 |
|---|---|---|
| **P0** | 在所有報酬數字旁標**漲停不可成交警示** + 把「GO」重新框架為「管線驗證、非 alpha 驗證」 | 目前文件最嚴重的過度宣稱 |
| **P0** | 算一個**最保守 net alpha 場景**(成本 0.5% + 漲停案例以 T+2 收盤、高 15-25% 進場)→ 看調整後超額報酬下限 | 「我們估最保守仍有 X%」比「會縮水」強十倍 |
| **P0** | 加**出場規則回測模擬**(如 T+30 出場 / -20% 停損),跟持有至 T+180 比 net P&L | 把訊號往「可執行策略」推的最低門檻 |
| P1 | beta/因子調整 + benchmark 對照 | 分離真 alpha |
| P1 | 二三階 lag 輪動策略化 | 系統獨有 edge、繞開漲停 |
| P2 | 多重比較校正 + look-ahead 檢查 | 學術級嚴謹,擋假 alpha |

---

## 7. 結論

> **PolyDig 在「找進場時機」有真材料(因果樹 + lag 是別人沒有的 edge),但現在是半套,而且帳面報酬有兩個致命水分:漲停買不到、未調 beta。** 金融專業的優化順序:① 誠實——每個漲幅數字標漲停警示、把 GO 降格為「管線驗證」、**親手算一個最保守 net alpha 下限**;② 補出場/部位/風控,讓訊號變可執行策略;③ 用「因果樹 lag」做二三階輪動(最可規模化、繞開漲停)。做完這三步,再談變現才有底氣;變現也請從**法規風險最低的 B2B 賣鏟子**起步。
