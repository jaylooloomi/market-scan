# Market Scan — VC 投資分析論點與改善建議

**分析日期**：2026-06-01
**視角**：早期（pre-seed / seed）創投盡職調查（diligence memo）
**標的**：Market Scan — 台股題材早期偵測，包成 Claude Code plugin（MIT、開源）
**分析基礎**：逐檔閱讀全部程式碼 + 設計 spec + Phase 0 驗證證據 + themes.json 資料資產
**Iteration**：1（Ralph loop — 後續會補競品數量化、TAM 試算細化）

> ⚠️ 我不是律師也不是你的財務顧問；以下「投顧法」段落是風險旗標，不是法律意見。

## 0. 一句話投資判斷

> **目前是一個「優秀的個人 alpha 工具 + 漂亮的作品集」，還不是一個「可投資的公司」。**
> 真正的價值主張（在無未來資訊下，即時從雜訊中辨識正確族群）**尚未被驗證**——而驗證它幾乎不花錢。
> **建議**：不以現狀投資；給一張「6 個月線上前瞻紀錄」的里程碑單，達標再談。這是一個典型的 **"feature, not a company yet"**，但**創辦人訊號（紀律、誠實、領域品味）很強**。

## 1. 這是什麼、市場在哪

- **產品**：每日掃 5 個訊號源（新聞 / 數據 / 價格 / 政策 / 路線圖）→ Claude agent 推因果樹（一/二/三階受益股 + lag）→ 比對歷史題材庫 → 出中文研究報告。定位是**研究助理，不自動下單**。
- **真正的洞見（North Star）**：「在事件還沒發酵、還有領先效果時抓到」。這個 framing 很對——它把產品和「落後指標型」的看盤工具區隔開。
- **市場**：
  - TAM 表面看是「台股散戶 + 主動投資人」（數百萬人），但
  - **可觸及市場（分銷現實）= Claude Code 使用者 ∩ 會交易台股的人 ∩ 信任 AI 選題材的人**——這是一個**很小的交集**。
  - 用 Claude Code marketplace 分銷很聰明（CAC 趨近 0、推理成本推給使用者自己的訂閱），但也把天花板鎖死在 Anthropic 平台用戶裡，且是平台依賴風險。

### 1.1 市場量級試算（order-of-magnitude；假設全部標明，非精確預測）

> 目的是判斷「是否 venture-scale」，非盡調級數據。台股母體用公開量級：證券開戶數 ~12M+、活躍散戶「數百萬」。

**透過 Claude Code 通路的 SAM（bottom-up 漏斗）**：

| 漏斗層 | 假設 | 量級 |
|---|---|---|
| 活躍台股散戶 | 公開量級 | ~6,000,000 |
| × 技術力足以裝 plugin + 設 `.env` + 申請 FinMind token | 0.5–2% | ~30,000–120,000 |
| × 真的會用「AI 選題材」工具 | 10–30% | **~3,000–36,000 可觸及用戶** |

→ **透過現有通路，可觸及市場是「數千～數萬人」的嗜好者規模（hobbyist-scale），不是 venture-scale。**

**營收量級（示意，非預測）**：免費 MIT plugin＝直接營收 $0（價值在個人 alpha / 作品集 / 招募）。即便轉付費、月費 NT$300–500、抓到上緣 ~1 萬付費 → 年營收**低個位數 NT$ 千萬級**＝可觀的 lifestyle business，但**達不到創投的規模曲線**。

**要做到 venture-scale，只有兩條路（都得離開現狀）**：
1. **脫離 Claude Code、做給一般散戶（SaaS/App）**：SAM 放大到數百萬，但**正面對撞 CMoney/XQ ＋ 踩投顧法**（§6）。
2. **B2B 賣鏟子**（資料/評測層給投顧/媒體/量化）：客單價高、避開對散戶薦股、TAM 質量較佳——本報告推薦的 wedge。

## 2. 核心賭注與證據：Phase 0 到底證明了什麼？

這是整份 memo 最關鍵的一段，也是創辦人**自己在 HANDOFF 裡誠實標註**的（加分）。

**Phase 0 證明了**：給定**後見之明手選的標的 + 手選的觸發日期**，這些著名題材（口罩/航運/AI/國防/矽光子）在事件主流化「之前」確實還有上漲空間。15 個測試點 → 6 強 / 5 弱 / 3 太晚 / 1 無效，4/5 case 有強領先 → GO。

**Phase 0 沒有證明（也就是真正的產品風險）**：
1. **即時辨識**：在當天的全部雜訊裡，無未來資訊地挑出「這條武漢肺炎新聞 → 口罩族群 → 康那香」。這是 Scout+Reviewer 的核心賭注，**完全未驗證**。
2. **精確率 / 召回率**：Phase 0 只看「選中的贏家有沒有領先空間」，沒有分母——系統每天會吐幾個假訊號？看到 100 條題材會錯報幾個?未知。
3. **存活者偏誤（survivorship bias）**：5 個 case、22 個歷史題材，全是「事後人盡皆知的大贏家」。沒有任何「看起來像題材但其實沒漲」的負樣本進入驗證。所以那個「73% 有領先 edge」**不是 out-of-sample 命中率**，而是「名人堂選手回測都很猛」的同義反覆。
4. **程式內的 backtest 也偏寬鬆**（見架構報告 M2）：`test_backtest_recall` 的成功判定近乎恆真，無法當作鑑別力證據。

> **VC 翻譯**：你證明了「這個市場（領先 alpha）存在」，還沒證明「你能穩定抓到它」。這兩件事的估值差距是 10 倍以上。

## 3. 護城河分析（Moat）

| 候選護城河 | 現狀評估 |
|---|---|
| **程式碼** | ❌ MIT 授權 + 開源，任何人可 fork。程式本身零護城河。 |
| **歷史題材庫（themes.json）** | ⚠️ 弱。22 個手curated題材、含完整因果樹——做得好，但任何資深台股分析師或一個 LLM 都能在一週內重建。它是**起跑資產，不是護城河**。 |
| **資料飛輪（missed_catch + verdicts 語料）** | ✅ 潛在最強——隨每日運行累積「我們漏抓了什麼、為什麼、後來怎樣」的負樣本與前瞻紀錄。**但目前是空的**（DB 還沒在線上持續跑）。這是唯一會隨時間複利的東西，現在 0 天資料。 |
| **創辦人領域品味** | ✅ 真實但不可規模化。因果樹的「孫子輩」洞見（宅經濟→ABF 載板）需要深台股 know-how。 |
| **分銷（marketplace 首發優勢）** | ⚠️ 先行者優勢有限，平台說改規則就改。 |

**結論**：護城河目前**只存在於「未來才會累積的前瞻紀錄」**。這也正好說明為什麼「現在就開始用 `--db` 每天跑」是最高槓桿的動作——它把唯一可複利的資產的計時器按下去。

## 4. 商業模式（最大的缺口）

- **現狀：沒有商業模式。** 免費、MIT、開源 plugin。沒有定價、沒有付費牆、沒有留存機制。
- **plugin 模式的隱藏優點**：推理成本（LLM）由使用者自己的 Claude 訂閱吸收 → **廠商端推理 COGS ≈ 0**。這是漂亮的單位經濟，但前提是你有「廠商」和「要收的錢」。
- **可走的方向（給 diligence 用）**：
  1. **付費訊號服務 / SaaS**：脫離 plugin，做有前瞻 track record 背書的訂閱報告。→ 撞上「投顧法」（見 §6）。
  2. **B2B / 賣鏟子**：把「事件→因果樹→受益股」當成**資料/評測層**賣給券商、投顧、財經媒體、量化團隊。避開對散戶薦股的法規風險，TAM 也更有質量。
  3. **開源獲客 + 商業版**：免費版打名氣，付費版給即時推播 / 個人化關注清單 / API。
  4. **純作品集 / 招募磁鐵**：不商業化，當作 think4u-tech 的能力展示與人才吸引——完全正當，但那就不是創投標的。
- **VC 觀點**：在選定 wedge 並證明有人願意付錢之前，這是 #1 未解問題。

## 5. 競爭與分銷

> 下表為**定性框架**（dimensions），非精確定價/用戶數——後者需 diligence 時實際查證，這裡不杜撰數字。

| 競品 / 類別 | 型態 | 領先 vs 落後 | 結構化因果樹（多階+lag） | 與 Market Scan 的關係 |
|---|---|---|---|---|
| **財經 M 平方 (MacroMicro)** | 總經/數據圖表訂閱 | 部分領先（總經指標） | ❌ 不做個股族群因果樹 | 精神最接近「數據先行」，但不薦台股個股族群——**最值得對標的定位參照** |
| **CMoney（籌碼K線/同學會）** | 散戶社群 + 籌碼/技術 | 落後型 | ❌ | 流量巨大但落後型，正是 Market Scan 想打的痛點 |
| **XQ 全球贏家（嘉實）** | 專業看盤/選股引擎 | 落後型（技術/籌碼） | ❌ | 專業付費工具，無事件→因果敘事 |
| **富果 Fugle** | API / 程式交易 | 中性（執行層） | ❌ | 偏「下單/回測基礎建設」，非「選題材」——潛在**互補**而非競爭 |
| **投顧老師 / LINE 群** | 人工薦股（持牌） | 同步/落後 | 人工、不結構化 | **法規對照組**：他們有投顧牌、Market Scan 沒有（見 §6） |
| **通用 LLM + 搜尋**（ChatGPT/Claude/Perplexity） | 對話式問答 | 可即時問「最近有什麼題材」 | ❌ 無歷史庫/無領先紀律/無持續監測 | **最大替代威脅**；Market Scan 的辯護＝結構化因果樹 + 歷史對應 + 每日領先監測 + track record |

- **競爭白地（Market Scan 真正的定位縫隙）**：「**領先型 + 結構化多階因果樹 + 歷史對應 + 研究助理（不薦股）**」這個四合一交集，目前市面上**沒有直接對手**——差異化故事成立。**但白地之所以是白地，部分原因是它窄、且踩在投顧法邊界上**（見 §6）。
- **最危險的不是現有台股工具，而是通用 LLM 的能力曲線**：當 Claude/GPT 內建更強的即時金融檢索，「問它最近有什麼題材」會吃掉輕度需求。Market Scan 唯一守得住的護城河仍是**累積的前瞻 track record + 漏抓語料**（見 §3）——再次指向「現在就開始每天用 `--db` 跑」。
- **分銷**：marketplace 首發是好故事（CAC≈0、推理成本由用戶訂閱吸收），但同時是 TAM 上限與平台依賴風險。

## 6. 關鍵風險（diligence red flags）

| 風險 | 嚴重度 | 說明 |
|---|---|---|
| **核心賭注未驗證**（即時偵測） | 🔴 高 | 見 §2。估值的 10x 差距就卡在這。 |
| **台灣投顧法規** | 🔴 高 | 對不特定公眾「提供分析意見或推介個股」在台灣屬《證券投資信託及顧問法》規管，可能需 SITCA/投顧執照。「研究助理、非投資建議」免責有幫助但**不是萬靈丹**，尤其一旦收費或公開大量分發。**任何商業化前務必先做法遵釐清。** |
| **存活者偏誤 / 無分母** | 🟠 中高 | GO 證據是後見之明選的贏家，缺假訊號率、缺精確率。 |
| **資料來源脆弱** | 🟠 中 | 大量靠免費 RSS/HTML/JSON（East Money、Google News RSS、TWSE OpenAPI、FRED）。任一改版即斷；SCFI/BDI/DRAM/法說逐字稿無穩定免費源（已誠實標 stub），代表 5 sensor 裡有幾個目前是半殘。 |
| **單一創辦人 / 巴士因子** | 🟠 中 | 領域品味集中在一人；HANDOFF 寫得好降低了交接風險，但團隊深度是創投必問。 |
| **平台依賴** | 🟡 中 | 綁 Claude Code / Anthropic 生態與 marketplace 政策。 |
| **無商業模式 / 無留存** | 🟠 中高 | 見 §4。 |
| **工程基礎建設不足** | 🟡 低（對投資）| 測試沒全綠、無 CI（見架構報告）——可快速修復，但反映「尚未為公開/規模做好準備」。 |

## 7. 要變得「可投資」，該做什麼（優先序）

1. **【最高槓桿、近乎免費】啟動線上前瞻紀錄**。立刻用 `market-scan-daily --db ./market-scan.db` 每天跑，持續 6-12 個月，累積時間戳化、out-of-sample 的 `verdicts` 與 `missed_catch`。然後**真的去量 spec §10 的線上指標**：平均領先天數 ≥14 天？月命中率（推薦後 90 天漲 ≥20%）≥30%？每日強訊號 ≤5 個？
   → 這一步把「不可證偽的故事」變成「有資料的論點」，是估值最大的解鎖。
2. **建立含負樣本的誠實回測**：把「看似題材但沒漲」的族群也放進評測，算出真實 precision/recall（修掉架構報告 M2 的恆真指標）。
3. **選一個 wedge 並驗證付費意願**：建議從「B2B 賣鏟子（資料/評測層給投顧/媒體/量化）」切入，避開對散戶薦股的法規風險，同時 TAM 質量較高。
4. **法遵前置**：在任何收費 / 公開大量分發前，找熟證券法的人釐清投顧執照需求與免責邊界。
5. **補工程基礎建設**（CI、測試全綠、套件可裝）——讓盡調工程師能在 10 分鐘內 clone→install→test 全綠，這本身就是一種訊號。
6. **把「資料飛輪」寫進敘事**：向投資人說明護城河是隨時間累積的前瞻語料，而非程式碼或那 22 個題材。

## 9. 一句話結論

> **Market Scan 證明了「領先 alpha 這個市場存在」，也展示了一位有紀律、誠實、有台股品味的建造者——但還沒證明「它能穩定抓到 alpha」，也還沒決定「要把它變成什麼生意」。** 好消息是：解鎖估值的那一步（每天用 `--db` 跑、累積前瞻紀錄）幾乎不花錢，而且現在就能開始。把計時器按下去，6 個月後我們再談一次。
